minGRU Architecture

Minimal Gated Recurrent Unit (minGRU)

Architecture Highlights

  • Only 39,690 parameters (vs ~120K for standard GRU)
  • Two gates: update gate (z) and reset gate (r)
  • Input shape: (batch_size, time_steps, features)
  • Output shape: (batch_size, num_classes)

Layer Structure

  • Input Layer → minGRU(64) → Dropout(0.3) → Dense(32) → Dropout(0.3) → Output
  • Learning rate: 0.001 with Adam optimizer
  • Batch size: 16 (optimized for small dataset)

The minGRU (Minimal Gated Recurrent Unit) architecture offers several advantages over traditional RNNs:

  • 39,690 trainable parameters (vs ~120K for standard GRU)
  • Only 2 gates instead of 3 in LSTM (reset and update gates)
  • Simpler computation flow for faster training and inference
  • Better performance on small datasets due to reduced parameter count

Challenges & Solutions

Current Challenges & Solutions

Data Limitations

  • Small dataset (599 windows)
  • Severe class imbalance (6 vs 216 samples)
  • High dimensionality (3 axes × 200 samples)
Solution:
  • SMOTE augmentation to balance classes
  • Class weights for gradient correction

Model Limitations

  • Training instability with small batch sizes
  • Overfitting indicators in validation performance
  • Limited generalization to unseen whales
Solution:
  • minGRU architecture (39,690 parameters)
  • Hierarchical classification approach (63.1%)
The optimized model achieves 57.2% validation accuracy on direct classification and 63.1% on hierarchical groups

The reported metrics are highly preliminary due to significant dataset limitations:

  • Performance will likely change substantially with more labeled data
  • Current focus is on establishing the classification pipeline framework
  • Awaiting additional annotations from Parks Lab for underrepresented classes
  • True validation will require a separate, well-balanced dataset

Implementation Details

Data Pipeline

  • HDF5 format for efficient data storage
  • Real-time SMOTE augmentation during training
  • Sequence standardization to 384 timesteps
  • Feature scaling and normalization

Training Approach

  • Adam optimizer with 1e-4 learning rate
  • Class weighting to handle imbalance
  • Early stopping on validation accuracy
  • Cross-validation with leave-one-out by whale
# minGRU implementation in PyTorch class MinGRUCell(nn.Module): def __init__(self, input_size, hidden_size): super(MinGRUCell, self).__init__() self.input_size = input_size self.hidden_size = hidden_size # Reset gate self.weight_xr = nn.Parameter(torch.Tensor(input_size, hidden_size)) self.weight_hr = nn.Parameter(torch.Tensor(hidden_size, hidden_size)) self.bias_r = nn.Parameter(torch.Tensor(hidden_size)) # Update gate self.weight_xz = nn.Parameter(torch.Tensor(input_size, hidden_size)) self.weight_hz = nn.Parameter(torch.Tensor(hidden_size, hidden_size)) self.bias_z = nn.Parameter(torch.Tensor(hidden_size)) # Candidate activation self.weight_xh = nn.Parameter(torch.Tensor(input_size, hidden_size)) self.weight_hh = nn.Parameter(torch.Tensor(hidden_size, hidden_size)) self.bias_h = nn.Parameter(torch.Tensor(hidden_size)) self.reset_parameters() def forward(self, x, h): # Reset gate r = torch.sigmoid(x @ self.weight_xr + h @ self.weight_hr + self.bias_r) # Update gate z = torch.sigmoid(x @ self.weight_xz + h @ self.weight_hz + self.bias_z) # Candidate activation h_tilde = torch.tanh(x @ self.weight_xh + (r * h) @ self.weight_hh + self.bias_h) # Final hidden state h_new = (1 - z) * h + z * h_tilde return h_new

Data Requirements & Next Steps

Current Limitations

  • Total dataset: ~20,000 frames
  • Only 50 expert annotations available
  • Extreme class imbalance (1-10 samples/class)
  • Insufficient data for proper validation
  • SMOTE augmentation cannot overcome core data scarcity

Required Improvements

  • Minimum 10 samples per behavior class
  • Dedicated validation set with ground truth
  • Multi-whale validation samples
